PyScript version 2022.09.1 was just released, and just as we lead dev Fabio Pliger said in proposing the versioning scheme:

"...An important aspect to keep in mind is that PyScript is still in its very early stages. So, we should highlight that the expectations should be that think can often break until we reach a level of maturity and stability."

And wow, are there a lot of new things in this version of PyScript. What's more, the default Pyodide runtime has been upgraded to the recently-released version 21.1, which itself provides many new features and improved functionality to PyScript.

I want to specificically highlight new features, breaking changes, and neat behind-the-scenes work. The full details of what's changed are captured in the PyScript Release Notes

PyScript

  • Multiple Runtimes Possible - type definition
  • Tailwind removed from build, CSS streamlined in PyScript.css
  • HTML tags are now escaped within PyScript tags
  • Improved error handling, including a fallback to error to the dev console if the error can't be displayed on the page.
  • Runtime API! Future support for additional runtimes! Currently only PyodideRuntime

Pyodide

Pyodide 21 Release Post and Change Log

  • Browser API wrappers for (#2456) for addEventListener and removeEventListener. Should avoid the need for create_proxy in addEventListener
  • Pyodide functions have been moved out of root pyodide package to individual modules (#2787, #2790)
  • Add pyodide.code.run_js to Python API. (#2426)
  • Implement slicing, +=, and extend for JsProxy objects (#2907)
  • Better error messages for removed stdlib libraries (#2973)
  • Correct handling of objects with null constructor (like modules) (#2520)
  • Improved building of binary wheels (release notes), including passing credentials, using pre-release and dev versions in micropip.install, installing Emscripten binary wheels.
  • Tons of new packages including opencv-python, ffmpeg, svgwrite, sqlite3, python-magic, and many more. (Full list)

Emscripten

Upgrading from v2.0.27 to v3.1.14

Intrastructure

  • Automatical build releases - releases pushed to tagged-release endpoint, every merge builds to unstable. Examples also rebuilt on
  • Woxtu adding type anotations many places

Testing

  • Playwright - loads HTML pages in the browser and checks that PyScript works as intended.
  • Jest - Testing whether Py-Script elements load correctly via TS/jS
  • PyTest - Test functionality in PyScript.py, Python things like Element, PyScript, ContextManager, etc.

Docs

  • "Trying before Installing" section
  • Diataxis framework defined
  • Improvements to CONTRIBUTING and TROUBLESHOOTING docs.

Minor

  • Lots of formatting adjustments
